# S&P 500 Momentum Strategy using Geometric Brownian Motion

> Masters-level financial data science project implementing a momentum-based algorithmic trading strategy using **Geometric Brownian Motion (GBM)**, **bootstrap parameter estimation**, **expected shortfall risk control**, **SQLite data engineering**, and **systematic backtesting**.

---

## Project Overview

This project develops a **momentum-based algorithmic trading strategy for the S&P 500 index**.

The strategy uses a **Geometric Brownian Motion (GBM) forecasting framework** to estimate the future distribution of prices based on historical momentum. Trading decisions are generated by comparing the forecasted price path against downside risk measured through **Expected Shortfall (ES)**.

The full workflow demonstrates an end-to-end quantitative research pipeline:

* market data ingestion from CSV
* storage in SQLite
* stochastic process calibration
* probabilistic forecasting
* downside tail-risk estimation
* position sizing
* historical backtesting
* strategy performance evaluation

This repository is designed to satisfy a **Masters-level university marking rubric** while also serving as a **portfolio-quality quant finance project**.

---

## Research Objective

The primary objective is to test whether **short-to-medium term momentum in the S&P 500** can be exploited using a statistically grounded stochastic model.

Specifically, the project answers:

> Can a GBM-calibrated momentum forecast generate risk-adjusted returns superior to passive exposure during the test period?

## Methodology

### 1) Data Engineering

Historical S&P 500 price data is loaded from `SP500.csv` and stored in SQLite.

Two tables are created:

* `prices` → stores daily market prices
* `positions` → stores strategy trades, holdings, and cash balance

This creates a **reproducible and queryable market data layer**.

---

### 2) GBM Calibration

The price process is modelled as:

```math
\mathrm{d}S_t = \mu S_t\,\mathrm{d}t + \sigma S_t\,\mathrm{d}W_t
```

Historical prices are transformed into **log returns**:

```math
r_t = \log\left(\frac{S_t}{S_{t-1}}\right)
```

A **bootstrap resampling procedure** estimates:

* first moment: $E[r]$
* second moment: $E[r^2]$

These moments are converted into GBM parameters:

```math
\sigma = \sqrt{\frac{E[r^2] - E[r]^2}{\Delta t}}
```

```math
\mu = \frac{E[r]}{\Delta t} + \frac{1}{2}\sigma^2
```

---

### 3) Forecasting

The calibrated GBM model forecasts the expected future index level over a configurable horizon.

Outputs include:

* expected future price
* lower confidence bound
* upper confidence bound

This creates a **probabilistic momentum signal rather than a binary trend rule**.

---

### 4) Risk Management — Expected Shortfall

The strategy uses **Expected Shortfall (ES)** as a downside tail-risk measure.

Unlike VaR, ES estimates the **average loss beyond the confidence threshold**, making it more suitable for risk-aware position sizing.

This is used to:

* penalise weak momentum signals
* reduce position size during elevated volatility
* improve tail-risk awareness

---

### 5) Position Sizing

Position sizes are determined by comparing:

* expected upside from the GBM forecast
* expected downside from ES

This creates a **risk-adjusted momentum allocation framework**.

---

### 6) Backtesting

The strategy is tested over the coursework evaluation window:

* **Train start:** rolling historical window
* **Test period:** `2020-06-01` to `2021-05-31`

Backtest outputs include:

* cumulative return
* daily returns
* best/worst days
* Sharpe ratio
* comparison to passive exposure
